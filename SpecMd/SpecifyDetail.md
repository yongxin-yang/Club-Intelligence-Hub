## 2025/12/19-1
现在后端的三个接口都能够正常开放了，你需要检查前端是否做好并给出前端的运行方式，实现最基本的AI client小功能（工作流）：
1. 用户在前端输入一个问题（例如“社团有哪些活动”）。
2. 前端将用户输入发送到 AI Gateway（`/chat` 接口）。
3. AI Gateway 调用 LLM 模型（如 OpenAI）并将用户问题传递给 MCP Server。
4. MCP Server 调用 Backend1 的 `/activities` 接口获取活动列表。
5. Backend1 返回活动列表给 MCP Server。
6. MCP Server 将活动列表整合到 LLM 模型的响应中。
7. AI Gateway 返回整合后的响应给前端。
8. 用户在前端看到包含活动列表的回答。
9. 用户继续输入问题（例如“社团的活动时间是多少”）。
10. 同上，此时用户最终在前端看到包含历史记录的回答。（前端也要有历史记录，可以记录到缓存中？反正是展示到对话框中）
注意，AI Gateway应该暴漏给前端一个接口用来告诉前端有哪些模型可以选择，而用户可以选择指定的有的模型。
